Да, теперь видно, что критичных различий почти нет.

И проблема, скорее всего, вообще не в b_user_access.

Вот эти отличия:

AE1
SND1
D1
SG8_A
SG8_E

— это нормально для руководителя/владельца проекта и отдела.

А отсутствие:

IU101 / U101

у второго пользователя — тоже нормально, потому что это уже его собственные коды:

IU139 / U139

То есть access codes у 101 выглядят корректными.

Тогда проблема почти наверняка в одном из двух:

1. Битый профиль пользователя

Очень частая проблема в коробке.

Особенно если:

пользователя давно создавали;

меняли тип;

переносили;

меняли отделы;

восстанавливали из бэкапа.



---

2. Сломанный relation cache/socialnetwork cache

Причём именно для аккаунта пользователя.
